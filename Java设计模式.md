# Java设计模式

### 设计模式介绍

设计模式是对软件设计中普遍存在（反复出现）的各种问题，所提出的解决方案。

**设计模式七大原则**

> - 单一职责原则
> - 接口隔离原则
> - 依赖倒转原则
> - 里式替换原则
> - 开闭原则
> - 迪米特法则
> - 合成复用原则

### 单一职责原则

**基本介绍：**

> 对类来说，即一个类应该只负责一项职责。假设类A 负责两个不同的职责：职责1和职责2。当职责1需求变更而改变A 时，可能造成职责2执行错误，所以需要将类A 的粒度分解为 A1，A2。

**注意事项和细节：**

> - 降低类的复杂度，一个类只负责一项职责
> - 提高类的可读性，可维护性
> - 降低变更引起的风险
> - 通常情况下，我们应当遵守单一职责原则，只有逻辑足够简单，才可以在代码级违反单一职责原则。只有类中方法数量足够少，可以在方法级别保持单一职责原则
